# Estilo de documentacion

## Reglas

- Escribir documentacion en espanol.
- Usar textos breves y accionables.
- Diferenciar claramente `IMPLEMENTADO`, `PLANIFICADO`, `PENDIENTE DE IMPLEMENTACION`, `PENDIENTE DE DEFINICION` y `NO CONFIRMADO`.
- No prometer funcionalidades inexistentes.
- No mezclar planes futuros con estado actual.
- No presentar decisiones pendientes como decisiones tomadas.
- Mantener la documentacion vigente alineada con el estado real del proyecto.
- No mencionar contextos externos al producto en documentacion permanente.

## ADRs

- Mantener numeracion secuencial.
- No borrar ADRs.
- Si una decision cambia, crear un nuevo ADR y marcar el anterior como `Reemplazado` o `Deprecado`.
