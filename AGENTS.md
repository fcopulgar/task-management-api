# Reglas para agentes

## Propósito

Este archivo define reglas obligatorias de trabajo para agentes que colaboren en este repositorio. Su objetivo es evitar cambios fuera de alcance, decisiones no trazadas y documentación desactualizada.

## Fuentes de verdad

1. `README.md`
2. `PLAN.md`
3. `AGENTS.md`
4. `docs/CURRENT_STATE.md`
5. `docs/requirements/`
6. `docs/DOMAIN_GUARDRAILS.md`
7. `docs/architecture/`
8. `docs/decisions/`
9. `docs/TECH_DEBT.md`

## Reglas de trabajo

- Trabajar solo dentro del alcance pedido.
- No adelantar fases futuras.
- No inventar requisitos.
- No inventar tecnologías.
- No inventar arquitectura.
- No implementar código si la tarea es documental.
- No modificar decisiones históricas sin crear un nuevo ADR.
- No borrar ADRs.
- No leer planes archivados salvo indicación explícita.
- Registrar solo deuda técnica real.
- Mantener toda la documentación en español.
- Marcar lo no definido como `PENDIENTE DE DEFINICIÓN`.
- Marcar lo no implementado como `PENDIENTE DE IMPLEMENTACIÓN`.
- Marcar lo no verificable como `NO CONFIRMADO`.

## Resumen obligatorio después de cada tarea

Siempre reportar:

- archivos modificados
- tareas del plan completadas
- validación realizada
- documentación actualizada
- deuda o pendientes detectados
