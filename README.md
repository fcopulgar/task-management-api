# task-management-api

## Estado inicial

Este repositorio está en fase de definición inicial. Todavía no tiene requisitos completos, tecnologías elegidas, arquitectura concreta ni implementación.

## Propósito del repositorio

El propósito de este repositorio es mantener una base documental y operativa para desarrollar el proyecto de forma incremental, trazable y mantenible mediante fases controladas.

## Estructura documental

- `PLAN.md`: plan activo del proyecto.
- `AGENTS.md`: reglas obligatorias para agentes.
- `docs/CURRENT_STATE.md`: estado real y vigente del proyecto.
- `docs/requirements/`: alcance y requisitos.
- `docs/architecture/`: documentación arquitectónica cuando exista una arquitectura definida.
- `docs/decisions/`: ADRs versionados y su plantilla.
- `docs/plans/`: planes archivados y futuros.
- `docs/standards/`: estándares de documentación, validación y cierre.

## Trabajo de futuros agentes

Todo agente debe leer primero las fuentes de verdad definidas en `AGENTS.md`, trabajar solo dentro del alcance solicitado y actualizar la documentación vigente al cerrar una tarea.

## Fuentes de verdad

Las fuentes principales son `README.md`, `PLAN.md`, `AGENTS.md` y `docs/CURRENT_STATE.md`.
